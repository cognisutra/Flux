Flux Documentation

A lightweight, zero-dependency scroll animation library with 99 animations across 25 categories.

Introduction

Flux lets you add smooth, performant animations to any element when it scrolls into view. Just add a data-aos attribute and you're done.

ⓘ Zero dependencies. ~6KB gzipped. Works in all modern browsers.

Installation

Copy these two files into your project:

<!-- flux.css — all animation definitions -->
<link rel="stylesheet" href="flux.css">

<!-- flux.js — scroll detection engine -->
<script src="flux.js"></script>

Flux auto-initializes on DOMContentLoaded. No manual setup required.

Usage

Add data-aos with any animation name to the element you want to animate:

<!-- Basic -->
<div data-aos="fade-up">Content</div>

<!-- With options -->
<div
  data-aos="zoom-in"
  data-aos-duration="800"
  data-aos-delay="200"
  data-aos-easing="ease-out-back"
  data-aos-once
>Animated content</div>
<!-- With presets -->
<div data-aos-preset="smooth">Preset animation</div>

<!-- With stagger (parent wraps children) -->
<div data-aos-stagger="100">
  <div data-aos="fade-up">Item 1</div>
  <div data-aos="fade-up">Item 2</div>
  <div data-aos="fade-up">Item 3</div>
</div>

<!-- Groups trigger together -->
<div data-aos="fade-up" data-aos-group="hero">Child A</div>
<div data-aos="fade-down" data-aos-group="hero">Child B</div>

<!-- Animate out when leaving viewport -->
<div data-aos="fade-up" data-aos-animate-out="true">Content</div>

<!-- Callbacks -->
<div
  data-aos="zoom-in"
  data-aos-on-start="myStartFunc()"
  data-aos-on-end="myEndFunc()"
>Content</div>

To customize global defaults, call Flux.init() with options:

<script>
  Flux.init({
    offset: 100,
    duration: 600,
    easing: 'ease-out-cubic',
    once: false,
    mirror: false,
    anchorPlacement: 'top-bottom',
    stagger: 50,
    debug: false,
    disable: false,
    presets: { myCustom: { animation: 'fade-up', duration: 1000 } }
  });
</script>

Data Attributes

Every aspect of an animation can be controlled per-element via data attributes.

Attribute Description Default
data-aos Animation name (e.g. fade-up, zoom-in) —
data-aos-duration Animation length in ms (50–3000) 600
data-aos-delay Stagger delay in ms (50–3000) 0
data-aos-easing Easing curve name ease-out-cubic
data-aos-offset Trigger offset in pixels 120
data-aos-once Animate only once (boolean — set any value) false
data-aos-mirror Re-trigger when scrolling back up false
data-aos-anchor-placement Trigger point: {el}-{viewport}. Options: top-bottom, center-center, bottom-bottom, top-center, bottom-center, etc. top-bottom
data-aos-preset Quick preset: subtle, smooth, dramatic, bouncy, energetic, reveal, focus, floaty —
data-aos-anchor CSS selector for the anchor element (triggers when anchor scrolls in) —
data-aos-group Group name — all elements in the same group animate together —
data-aos-stagger Set on a parent to stagger children by N ms each 0
data-aos-animate-out Remove the animate class when the element scrolls back out of view false
data-aos-on-start JavaScript callback function name when animation starts —
data-aos-on-end JavaScript callback function name when animation ends —

API Methods

Flux.init(options)

Initialize or re-initialize the library with custom global defaults. Auto-called on DOM ready if [data-aos] elements exist.

Flux.refresh()

Re-scan the DOM for new [data-aos] elements. Useful after dynamically adding content.

Flux.replay()

Reset and re-trigger all visible animations. Useful for preview or demo scenarios.

Flux.reset()

Remove aos-animate from all tracked elements without re-triggering.

Flux.destroy()

Clean up all observers and remove event listeners.

Flux.debug(show)

Toggle the debug overlay. Pass true or false to force show/hide, or call with no argument to toggle. Keyboard shortcut: Ctrl+Shift+D (when debug is enabled in init options).

Advanced Features

Presets

Built-in presets let you skip individual attributes and get a complete animation look with one attribute:

<!-- Each preset sets animation, duration, and easing in one go -->
<div data-aos-preset="smooth">Smooth entrance</div>
<div data-aos-preset="dramatic">Dramatic entrance</div>
<div data-aos-preset="bouncy">Bouncy entrance</div>

Stagger

Add data-aos-stagger to a parent element to automatically delay each child animation by N ms:

<!-- Each child animates 100ms after the previous one -->
<div data-aos-stagger="100">
  <div data-aos="fade-up">Item 1</div>
  <div data-aos="fade-up">Item 2</div>
  <div data-aos="fade-up">Item 3</div>
</div>

Animation Groups

Elements sharing the same data-aos-group value will all animate simultaneously once any one of them is triggered:

<div data-aos="fade-up" data-aos-group="hero">Title</div>
<div data-aos="fade-down" data-aos-group="hero">Subtitle</div>

Anchor Reference

Trigger an animation based on a different element's position using data-aos-anchor:

<!-- This element animates when #trigger scrolls into view -->
<div id="trigger"></div>
<div data-aos="fade-up" data-aos-anchor="#trigger">Animated</div>

Animate Out

By default, elements keep their animated state once triggered. Add data-aos-animate-out to remove the class when the element scrolls back out of view:

<div data-aos="fade-up" data-aos-animate-out>Fades back out when leaving viewport</div>

Callbacks

Execute JavaScript functions when an animation starts or ends. The value is a function name or inline expression:

<script>
  function onEnter() { console.log('Entered viewport!'); }
  function onLeave() { console.log('Animation complete!'); }
</script>
<div data-aos="fade-up"
     data-aos-on-start="onEnter"
     data-aos-on-end="onLeave">Content</div>

Responsive Disable

Globally disable Flux on specific screen sizes via the disable init option or set true to disable entirely:

Flux.init({
  disable: 'mobile'                   // disable below 768px
  // disable: 'tablet'                // disable below 1024px
  // disable: 'mobile,tablet'         // both
  // disable: 'desktop'               // disable above 1024px
  // disable: true                    // disable entirely
});

Reduced Motion

Flux respects the prefers-reduced-motion: reduce media query automatically. When detected, all durations and delays are set to 1ms for an instant reveal. You can force-enable animation for these users by passing reducedMotion: true to Flux.init().

Debug Overlay

Enable the debug overlay to see which elements Flux is tracking and their animation state. Toggle with Ctrl+Shift+D:

Flux.init({ debug: true });

// Or toggle at runtime:
Flux.debug();         // toggle on/off
Flux.debug(true);    // show
Flux.debug(false);   // hide
